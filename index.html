01
🤖

¿Qué es un Agente?

Un agente es una entidad autónoma —de software o física— que percibe su entorno, procesa información, toma decisiones y ejecuta acciones para alcanzar un objetivo específico. A diferencia de un simple programa, el agente actúa de forma proactiva, adaptándose al contexto sin requerir instrucción humana en cada paso. Puede aprender de sus resultados y mejorar su desempeño con el tiempo.

agentes
02
🧠

¿Qué es un Multiagente?

Un sistema multiagente está compuesto por varios agentes autónomos que interactúan, se coordinan y colaboran para resolver problemas complejos que individualmente no podrían abordar. Cada agente puede tener especialidades distintas; la arquitectura multiagente distribuye el trabajo, optimiza procesos y permite soluciones más robustas mediante cooperación y comunicación entre sus componentes.

agentes
03
⚙️

¿Para qué sirve un Multiagente?

Los sistemas multiagente sirven para automatizar flujos complejos de extremo a extremo: dividen un problema grande en subtareas, asignan cada una a un agente especializado y coordinan los resultados de forma sincrónica o asíncrona. Son ideales para gestión de proyectos, análisis paralelo de datos, simulaciones y cualquier proceso que requiera múltiples puntos de vista o competencias simultáneas.

agentes
04
🦾

¿Qué es Droid?

En el contexto de IA moderna, Droid es un framework o plataforma diseñada para construir y desplegar agentes o sistemas multiagente que ejecutan tareas de forma autónoma —especialmente en entornos de desarrollo y código. También puede referirse a un robot físico inteligente o a un sistema operativo móvil (Android). Su característica central es la capacidad de orquestar múltiples agentes para completar proyectos complejos.

herramientas
05
🎭

¿Qué es un Fake Bot?

Un fake bot es un programa automatizado diseñado para simular ser un usuario humano real con fines engañosos o fraudulentos. Puede adoptar una identidad falsa, generar actividad artificial (likes, comentarios, interacciones), difundir información errónea o manipular sistemas. Se diferencia de un agente legítimo en que su objetivo central es el engaño, no la asistencia genuina.

06
📦

¿Para qué sirve un Repositorio?

Un repositorio es un lugar centralizado donde se almacena, organiza y controla la historia de un proyecto digital. Permite guardar cada versión del código o documentos, registrar quién hizo qué cambio y cuándo, colaborar con el equipo sin sobrescribir el trabajo de otros, y revertir a versiones anteriores en caso de errores. Es la base del trabajo colaborativo moderno en desarrollo de software y proyectos de IA.

herramientas
07
🐙

¿Qué es GitHub?

GitHub es la plataforma en línea más popular para alojar, gestionar y colaborar en proyectos de software mediante Git, el sistema de control de versiones distribuido. Funciona como una red social del código: permite crear repositorios, hacer seguimiento de cambios, trabajar en ramas paralelas, revisar código entre pares y lanzar versiones. También soporta integración continua, documentación y automatización de flujos de trabajo.

plataformas
08
🔄

¿Cómo funciona GitHub?

El flujo básico de GitHub es: clonas el repositorio a tu equipo local → realizas cambios en una rama → haces commit (registras los cambios con un mensaje) → subes (push) al repositorio remoto → abres un Pull Request para que otros revisen → se fusiona (merge) la rama al proyecto principal. Cada paso queda registrado con autor, fecha y descripción, garantizando trazabilidad total.

09
⚖️

¿Diferencia entre Claude y Droid?

Claude es un modelo de lenguaje conversacional (IA generativa) creado por Anthropic: razona, redacta, analiza y responde preguntas complejas. Droid es un framework o herramienta orientada a la ejecución autónoma de tareas —especialmente código— mediante agentes. En síntesis: Claude es el "cerebro que piensa y razona"; Droid es el "operador que actúa y ejecuta". Pueden complementarse en una arquitectura donde Claude razona y Droid orquesta la ejecución.

¿Qué es Open Cloud?

Open Cloud es un concepto de infraestructura en la nube basado en estándares abiertos e interoperables, que permite a organizaciones mover aplicaciones y datos entre distintos proveedores sin quedar atrapadas en uno solo (evitando el vendor lock-in). Ofrece servicios de cómputo, almacenamiento y conectividad con total flexibilidad, fomentando la portabilidad, la soberanía de datos y la reducción de costos.

plataformas
11
💬

¿Qué es una Sesión de Uso en Ollama?

Una sesión de uso en Ollama es el período activo durante el cual un modelo de IA corre localmente en tu máquina, mantiene el contexto de la conversación en memoria y responde consultas sin necesidad de internet. Mientras la sesión está abierta, el modelo recuerda el hilo de la charla. Al cerrarla, el contexto se pierde —salvo que se exporte— y los recursos del sistema son liberados.

herramientas
12
📊

¿Qué es un Sistema Predictivo de Agentes?

Un sistema predictivo de agentes es una arquitectura donde uno o varios agentes de IA analizan datos históricos y en tiempo real para anticipar eventos, comportamientos o riesgos futuros y tomar decisiones proactivas en consecuencia. A diferencia de un agente reactivo (que responde a lo que ya ocurrió), el sistema predictivo actúa antes de que el problema se materialice, optimizando recursos y reduciendo la incertidumbre.

agentes
13
🎓

¿Para qué sirve capacitar con un Agente?

Usar un agente para capacitación permite ofrecer aprendizaje personalizado, interactivo y disponible 24/7. El agente puede simular escenarios reales, evaluar conocimientos, dar retroalimentación inmediata, adaptar el contenido al nivel del usuario y escalar la formación a toda una organización sin costo proporcional. Estandariza el aprendizaje eliminando sesgos del instructor y acelera la adopción de nuevos conocimientos.

ia
14
🛠️

¿Qué es un Skill (habilidad)?

Un skill en el contexto de agentes de IA es una capacidad específica —o herramienta— que un agente puede invocar para ejecutar una tarea concreta: buscar en la web, leer un archivo, escribir código, enviar un correo, llamar a una API, etc. Los skills son los "superpoderes" del agente: amplían lo que puede hacer más allá de la generación de texto puro, conectándolo con el mundo real y sistemas externos.

herramientas

🌐

Crear un Glosario en HTML

Un glosario en HTML es una página web que organiza y presenta definiciones de términos de forma visual, interactiva y accesible desde cualquier dispositivo con navegador. Permite búsqueda en tiempo real, filtros por categoría, diseño responsive y puede publicarse en línea de forma gratuita para compartirse con un enlace. Es la evolución digital de los glosarios impresos: dinámico, actualizable y colaborativo.

🚀 ¿Cómo publicar este glosario online?

Paso a paso para tener un link público para compartir

1
Crear cuenta en GitHub

Ve a github.com → Sign Up → completa el registro gratuito.

2
Crear un nuevo repositorio

Clic en "New repository" → nómbralo glosario-ia-agentes → marca "Public" → Create.

3
Subir el archivo HTML

En el repositorio → "Add file" → "Upload files" → arrastra glosario_ia_agentes.html → renómbralo como index.html → Commit changes.

4
Activar GitHub Pages

Settings → Pages (menú izquierdo) → Source: "Deploy from a branch" → Branch: main → Carpeta: / (root) → Save.

5
¡Tu link está listo!

En ~2 minutos aparecerá tu URL pública: https://TU-USUARIO.github.io/glosario-ia-agentes/ — ¡compártela!

💡 Alternativa rápida: También puedes usar Netlify Drop — arrastra el HTML a netlify.com/drop y obtén un link en segundos, sin crear cuenta.
